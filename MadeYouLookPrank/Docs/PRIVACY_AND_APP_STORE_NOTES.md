# Privacy and App Store Notes

## Privacy

This project is for personal, consent-based prank use only.

- The app tells the user before capture that a selfie will be taken.
- The app captures one front-camera photo only after the user taps Start.
- Photos are stored locally in the app's private Application Support folder.
- There is no network transmission, analytics SDK, advertising SDK, cloud storage, or remote logging.
- Exporting to the Photos library only happens after the owner chooses a gallery export action.

## Why This Is Not Disguised as Tinder

Using the actual Tinder name or logo would create trademark and impersonation problems. This project uses an original flame mark and the display name `FlameMatch`.

## Why This Does Not Secretly Capture Photos

Covert camera capture, deceptive permission copy, suppressed shutter indicators, and hidden collection of someone else's face are privacy-invasive. The finished app keeps the prank reveal but makes capture explicit and local-only.

## App Store Restrictions

Even this consent-based prank should be reviewed carefully before any distribution. A deceptive version that impersonates another app, tricks people into camera access, or captures faces without informed consent would likely violate App Store privacy and review rules and may violate local law.

This project is intended for sideloading to your own device for personal testing, not App Store release.
